# আপনার API কী সেট করুন

আপনার Claudin.io কী **একবার** একটি পরিবেশ পরিবর্তনশীল হিসেবে সেট করুন এবং এই গাইডের প্রতিটি টুল এটি পুনরায় ব্যবহার করতে পারে — প্রতিটি ক্লায়েন্টে হাতে করে পেস্ট করার প্রয়োজন নেই।

আপনার `sk-...` কীটি [ড্যাশবোর্ড](https://claudin.io/dashboard) থেকে নিন (দেখুন [আপনার অ্যাকাউন্ট তৈরি করুন](account.md)), তারপর এটি আপনার শেল প্রোফাইলে যোগ করুন যাতে এটি প্রতিটি নতুন টার্মিনালে উপলব্ধ থাকে।

## macOS / Linux

=== "zsh (macOS-এ ডিফল্ট)"

    ```bash
    echo 'export CLAUDINIO_API_KEY="sk-..."' >> ~/.zshrc
    source ~/.zshrc
    ```

=== "bash"

    ```bash
    echo 'export CLAUDINIO_API_KEY="sk-..."' >> ~/.bashrc
    source ~/.bashrc
    ```

`sk-...` কে আপনার আসল কী দিয়ে প্রতিস্থাপন করুন। আপনি কোন শেলে আছেন তা নিশ্চিত না? `echo $SHELL` চালান।

## যাচাই করুন

```bash
echo $CLAUDINIO_API_KEY
```

আপনার কীটি আবার প্রিন্ট হওয়া দেখা উচিত। যদি এটি খালি থাকে, একটি নতুন টার্মিনাল খুলুন অথবা উপরের `source` কমান্ডটি পুনরায় চালান।

## কেন এটি সাহায্য করে

[আপনার টুল সংযুক্ত করুন](../clients/opencode.md) বিভাগের প্রতিটি **দ্রুত সেটআপ** স্ক্রিপ্ট `$CLAUDINIO_API_KEY` পড়ে, তাই এটি একবার এক্সপোর্ট করা হলে আপনি যেকোনোটি সেভাবেই চালাতে পারেন — প্রতিস্থাপনের জন্য কোনো `YOUR_API_KEY` নেই। যে টুলগুলি সরাসরি পরিবেশ পরিবর্তনশীল পড়ে (Codex-এর `env_key`, যেকোনো OpenAI-সামঞ্জস্যপূর্ণ CLI) সেগুলিও স্বয়ংক্রিয়ভাবে এটি গ্রহণ করে।

!!! warning "আপনার কীটিকে পাসওয়ার্ডের মতো আচরণ করুন"
    এই কী যার কাছে থাকবে সে আপনার প্ল্যানের বাজেট খরচ করতে পারে। আপনার `~/.zshrc` / `~/.bashrc` একটি পাবলিক রিপোজিটরিতে কমিট করবেন না। যদি কী লিক হয়, ড্যাশবোর্ডে এটি প্রত্যাহার করুন এবং একটি নতুন কী এক্সপোর্ট করুন।
